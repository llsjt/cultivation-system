# 需求分析 v1.6

版本：v1.6
状态：草稿
整理日期：2026-06-01
前置版本：`../v1.5/需求分析.v1.5.md`
对应实施方案：待补充 `实施方案.v1.6.md`

## 分析结论

v1.6 的核心目标是优化境界提升的权重分配，让“项目境界”更贴近一个学习方向的真实积累。新的需求口径是：境界提升主要看这个方向一共有多少资料，再按照每份资料的权重和实际学习时间提升境界。一般情况下，一个方向的学习资料不低于 3 个更能形成稳定评价，但这不是强制门槛。

这里的“方向”对应当前系统里的项目；“资料”对应该项目下的资源。v1.5 已把境界系统定位为项目级轻量成就反馈，v1.6 不扩大成游戏系统，只调整境界提升依据：从固定四维公式，收敛到“资料规模、资料代表性、有效学习时间”三件事。

当前实现中，`src/shared/realm.ts` 主要使用核心掌握、试炼掌握、复盘证据和稳定性计算道基评分；资源已有 `mastery_weight`，学习记录已有 `duration_minutes`。这说明数据基础已经具备，但评分主线还没有按“资料数量 + 权重 + 学习时间”表达。

v1.6 应把境界提升权重从“固定公式触发突破”升级为“方向资料池的加权学习积累”。资料越多、代表性越高、有效学习时间越充分，境界提升越稳；资料过少、权重失衡或学习时间不足时，应给出清楚的补齐建议。

## 范围原则

- 只优化项目境界的评分和突破判定口径，不新增角色等级、战力、排行榜、抽卡、技能树等游戏系统。
- 方向的学习资料数量一般建议不低于 3 个；少于 3 个时可以继续学习、记录和计算境界，但评价可信度应降低，并提示补充资料。
- 境界提升主要由资料规模、资料权重、有效学习时间决定；进度百分比、复盘证据和稳定性只作为辅助解释或质量校验。
- 学习时间必须防刷：单份资料、单次记录、单日累计都需要有合理上限或衰减，避免挂时长直接突破。
- 不删除现有境界字段，不做破坏性数据迁移；已有项目应按新口径重新计算展示。
- 权重模型必须能被普通用户理解为学习反馈，而不是黑箱数值系统。

## P0 需求分析

### RA16-01 境界提升缺少资料规模基线

问题：当前突破条件主要围绕核心资料、试炼资料、道基总分和出关记录，缺少“这个方向有多少学习资料支撑评价”的规模判断。一个方向只有 1 到 2 份资料时，即使这些资料进度很高，也只能说明该方向已有集中学习，不应被解释成资料池已经很完整。

影响：资料很少的项目如果完全按同一口径评价，境界反馈可能变成对单份资料的完成反馈，而不是对一个方向的阶段性评价；但如果把 3 个资料写死为硬门槛，又会误伤轻量方向或早期探索方向。

需求判断：v1.6 应把资料数量作为境界提升的规模基线，而不是强制门槛。一般方向建议不低于 3 个有效学习资料；低于 3 个时仍可按权重和学习时间提升境界，但需要降低评价可信度，并优先提示补充资料。

建议规则：

- 有效学习资料指参与境界计算的资料，不包含纯参考、不参与学习积累的资料。
- 少于 3 个有效学习资料时，境界面板应提示“资料规模偏少”，优先建议补充资料，但不直接阻断境界计算。
- 3 个资料是默认经验线，不是硬性门槛，也不是鼓励堆资料；超过经验线后，仍要看权重和学习时间。
- 同源重复资料可以计入资料池，但其境界贡献应衰减，避免重复堆同一内容。

验收口径：

- 方向少于 3 个有效学习资料时，仍可计算境界积累，但应显示资料规模偏少的提示。
- 方向达到 3 个有效学习资料后，资料规模不再成为主要风险提示，系统主要看权重和学习时间。
- 界面能说明“还需补充几份资料”而不是只显示道基不足。
- 纯参考资料不会被用户误以为能撑起境界提升。

### RA16-02 资料权重应表达方向代表性

问题：`mastery_weight` 当前取值为 1 到 5，但需求没有明确说明它表达的是资料重要性、难度、耗时还是优先级。若含义不清，用户可能把“难度高”“最近想学”“资料很长”都填成高权重，导致境界分数失真。

影响：境界提升会被主观偏好或资料长度误导。真正代表这个方向能力的资料如果权重过低，系统会低估关键学习；重复、边缘或补充资料如果权重过高，系统会高估积累。

需求判断：v1.6 需要把资料权重定义为“这份资料对当前方向境界的代表性”。权重越高，表示这份资料越能代表该方向的核心能力或阶段性成果。

建议规则：

- `1`：补充资料，只轻微影响境界。
- `2`：普通资料，默认学习材料。
- `3`：重要资料，会明显影响境界积累。
- `4`：关键资料，学习不足时应明显拖慢境界提升。
- `5`：基准资料，代表该方向核心能力证明，应谨慎使用。
- 权重不是难度、耗时或喜好；高权重资料仍需要有效学习时间支撑。

验收口径：

- 新建和编辑资料时，权重文案能说明“方向代表性”含义。
- 高权重资料学习时间不足时，用户能理解为什么境界提升被拖慢。
- 低权重补充资料不会因为数量多而压过关键资料。
- 权重 5 的使用应在文案上提示谨慎，避免所有资料都被设为最高权重。

### RA16-03 学习时间应成为境界提升主驱动

问题：当前境界计算主要看资料进度百分比和日志证据类型，没有把学习时间作为主驱动。系统已有 `duration_minutes`，但它更多用于记录展示，没有直接成为境界提升的核心依据。

影响：用户花了大量时间学习关键资料，却可能因为进度或证据类型不匹配而反馈不足；反过来，快速填写进度和记录也可能带来过高境界收益。

需求判断：境界提升应主要来自加权有效学习时间。每份资料的境界贡献应由“资料权重 × 有效学习时间”决定，再结合资料数量和合理上限计算方向积累。

建议规则：

- 有效学习时间优先来自受控打开产生的 pending 时长，其次使用用户手动填写的出关时长。
- 单次学习时长应有上限，避免一次记录异常放大贡献。
- 单份资料应有阶段上限，避免只靠一份资料无限提升整个方向境界。
- 单日总贡献应有合理上限或衰减，避免补录大量时长直接突破。
- 没有填写学习时间时，可以保存学习记录，但境界贡献应有限或提示补时长。

验收口径：

- 两个方向资料数量和权重相同，学习时间更充分的一方境界积累更高。
- 高权重资料同等学习时间下贡献高于低权重资料。
- 单份资料学习时间再高，也不能完全替代方向中其他资料的最低积累。
- 异常长时长不会让项目一次性跨越不合理境界。

### RA16-04 资料数量、权重和时间需要组合成可解释模型

问题：只看资料数量会鼓励堆资料；只看权重会鼓励把所有资料设成高权重；只看时间会鼓励挂时长。三者必须组合，否则境界提升会偏向某一种可刷行为。

影响：如果模型不可解释，用户只会猜“怎么刷分”。如果模型过于复杂，又会让境界系统压过学习工具本身。

需求判断：v1.6 需要定义一个简单、可解释的组合口径：资料数量决定方向基础，资料权重决定代表性，学习时间决定真实投入。三者缺一不可。

建议模型口径：

- 资料规模因子：方向有效学习资料接近或达到 3 个时，评价可信度更高；少于 3 个时降低可信度但不硬阻断。
- 单资料贡献：按 `资料权重 × 有效学习时间` 计算，并受单资料上限约束。
- 覆盖度约束：有效学习资料中应有多份产生学习时间，不能只集中在一份。
- 同源衰减：同一主题的重复资料贡献逐步降低。
- 境界层数：可由当前阶段累计贡献映射到 1 到 9 层。
- 境界突破：需要资料数量达标、总贡献达标、覆盖度达标。

验收口径：

- 用户能从境界面板看出：资料还差多少、关键资料学了多久、总积累差多少。
- 资料多但学习时间不足时，提示“资料池已具备，学习积累不足”。
- 学习时间高但只集中在一份资料时，提示“覆盖资料不足”。
- 权重高但未学习时，不产生虚高贡献。

### RA16-05 突破诊断需要按新口径重排

问题：当前突破失败提示偏向核心掌握、试炼掌握、道基评分和出关记录。v1.6 改为资料数量、资料权重和学习时间后，诊断顺序也必须调整，否则用户仍会按旧模型补错方向。

影响：用户可能看到“道基不足”却不知道是资料少、关键资料权重高但时长低，还是学习时间集中在单一资料。诊断不能指导下一步，就会变成噪音。

需求判断：突破诊断应按新模型展示硬条件和建议项：先看资料池是否足够，再看加权学习时间是否足够，再看覆盖度和异常值。

建议诊断顺序：

1. 方向有效学习资料数量是否偏少，是否低于一般建议的 3 个。
2. 关键权重资料是否已有有效学习时间。
3. 加权学习时间总量是否达到当前境界要求。
4. 学习时间是否覆盖多份资料，而不是集中在一份。
5. 是否存在异常长时长、重复同源资料或未计入资料。

验收口径：

- 突破失败时能优先提示最可执行的补齐动作。
- 资料数量偏少时，不再优先提示复盘或稳定性，而是提示资料规模偏少和可补充方向。
- 时间不足时，能指出是总时长不足还是关键资料时长不足。
- 只显示第一条瓶颈时，也能进入完整诊断信息。

### RA16-06 缺少新权重模型的回归样例

问题：当前测试覆盖了可突破、不可突破、同源衰减和层数边界，但没有覆盖资料数量、资料权重、学习时间、时间集中度和异常时长等关键样例。

影响：后续优化公式时，开发者只能靠局部测试判断是否通过，无法判断境界节奏是否整体符合“资料池 + 权重 + 时间”的需求。

需求判断：v1.6 必须先定义回归样例，再调整公式。样例应覆盖“应该通过”“应该卡住”“边界可解释”三类。

建议样例：

- 2 个资料、学习时间很高：可以产生境界积累，但提示资料规模偏少，评价可信度低于 3 个以上资料的方向。
- 3 个资料、总时间不足：不可突破，提示学习积累不足。
- 3 个资料、时间集中在 1 个资料：不可突破或接近突破，提示覆盖不足。
- 3 个资料、权重和时间分布合理：可提升层数，达到门槛时可突破。
- 高权重资料未学习、低权重资料时间很多：不可突破或贡献有限，提示关键资料不足。
- 单次异常时长很高：贡献被上限约束，不会一次性跨越不合理境界。
- 同源重复资料很多：贡献衰减，不会线性堆高。

验收口径：

- 每次改动境界权重，都必须运行覆盖上述样例的测试。
- 测试断言不只检查 `can_breakthrough`，也检查资料数量、加权时间、覆盖度和瓶颈文案。
- 样例数据使用固定时间，避免当前日期影响结果。

## P1 需求分析

### RA16-07 权重解释需要进入用户界面

问题：即使模型合理，如果界面只显示“道基评分 78 / 需巩固”，用户仍无法理解下一步该补资料、学关键资料，还是补学习时长。

需求判断：v1.6 应在境界面板中展示权重构成，但保持克制，不把界面做成复杂数值面板。

建议展示：

- 当前方向有效资料数量，例如 `2/3` 或 `5 份有效资料`。
- 加权学习时间总量和当前境界所需时间。
- 贡献最高和最不足的 1 到 3 份资料。
- 学习时间覆盖情况：是否集中在单一资料。
- 下一步建议：补资料、学习高权重资料、补关键资料时长。

验收口径：

- 用户能在 10 秒内看出“补资料、学哪份资料、还差多少时间”。
- 权重说明不遮挡首页继续学习和出关记录主路径。
- 不展示复杂公式作为主要 UI，只展示可行动信息。

### RA16-08 权重调整需要兼容已有项目

问题：权重模型改变后，已有项目的道基评分、境界层数和可突破状态可能发生变化。如果没有过渡说明，用户会觉得进度被系统“偷改”。

需求判断：v1.6 需要定义兼容策略。优先选择重新计算展示，不做数据回滚；但必须让用户知道这是评分口径优化，不是学习记录丢失。

验收口径：

- 已有资料、学习记录、突破记录不被删除或重写。
- 已有学习记录中的 `duration_minutes` 能参与新口径；缺失时按低贡献或零贡献处理，并给出提示。
- 权重口径调整后，项目境界状态能重新计算并稳定展示。
- 如果分数变化明显，界面或版本说明能解释“境界评分口径已优化”。

## 不做事项

- 不开放用户自定义任意评分公式。
- 不新增角色属性、战斗力、排行榜、抽卡或境界树。
- 不把学习时间变成强制打卡或刷时长任务。
- 不为了权重优化删除已有学习记录或突破记录。
- 不用单一资料完成度替代整个方向的境界提升。
- 不把所有资料都强制要求同一种类型；3 个资料只是一般建议，不作为强制资料数量门槛。

## 开放问题

1. 低于 3 个资料时，评价可信度应如何降低：仅提示、降低贡献系数，还是限制突破但不限制层数提升？
2. 每个境界或每一层需要多少加权学习时间，是否随境界提升递增？
3. 单份资料、单次记录、单日累计的时间上限应分别是多少？
