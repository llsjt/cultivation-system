# 实施方案 v1.5

版本：v1.5
状态：草稿
整理日期：2026-05-31
对应需求分析：`需求分析.v1.5.md`

## 1. 版本目标

v1.5 的目标是把现有自用 MVP 收敛成“每天敢用”的学习闭环：

1. 打开应用后，首页首屏能立刻回答“继续学什么、上次学到哪、完成多少、下次做什么”。
2. 出关记录默认 30 秒内可完成，保存后能清楚看到进度变化和下次目标。
3. 项目创建、删除、刷新、待记录会话和全局资源库不再出现选中错位、漏数据或弱契约问题。
4. 境界系统正式降维定义为“项目级轻量成就反馈”，不继续扩成独立游戏系统。
5. 前端拆分、样式分层、IPC 硬化只服务本轮闭环稳定，不做全量重写。

## 2. 不做事项

- 不做角色养成、战力、排行榜、抽卡、成就墙扩展。
- 不做完整设计系统重写，不替换 React/Electron/Vite 技术栈。
- 不做所有 IPC 一次性重构，只覆盖本轮关键命令和新增命令。
- 不做全量 CSS 视觉改版，只做局部分层和关键布局稳定。
- 不做发布级完整 WCAG 自动化覆盖，只补核心路径可访问性。
- 不做云同步、导入导出、AI 总结、复习提醒和完整日志管理页。

## 6. M1 首页与状态底座

### PLAN-v1.5-004 拆出 overview 和选中项目状态

目标：修复项目刷新竞争，并给首页恢复信息打底。

改动范围：

- `src/renderer/App.tsx`
- 新增 `src/renderer/hooks/useOverviewQuery.ts`
- 新增 `src/renderer/hooks/useSelectedProject.ts`
- 新增 `src/renderer/hooks/useResourceActions.ts`
- 可选新增 `src/renderer/features/home/HomeView.tsx`

执行步骤：

1. 把 overview/enums 加载从 `App.tsx` 抽到 `useOverviewQuery`。
2. 把 `selectedProjectId`、`projectDetail`、`projectCultivation`、`loadProject` 抽到 `useSelectedProject`。
3. 把资源创建、编辑、删除、继续学习、打开资料相关 mutation 抽到 `useResourceActions`。
4. `useResourceActions` 不直接持有全局 overview，只通过回调触发刷新、Toast、确认框和 pending 处理。
5. 将 `refresh()` 改成 `refresh(nextSelectedProjectId?: string | null)`。
6. refresh 后用最新 `data.projects` 校验目标 ID：
   - 目标存在：选中目标。
   - 目标不存在：选中第一个项目。
   - 无项目：清空详情和境界状态。
7. 创建项目后调用 `refresh(project.id)`。
8. 删除项目后调用 `refresh(null)` 或 `refresh(fallbackId)`。

验收：

- 创建项目后，新项目立即选中。
- 删除当前项目后，不再请求已删除项目。
- 手动刷新保留当前有效项目。
- `App.tsx` 不再直接维护所有项目详情加载细节。
- `App.tsx` 不再直接承载资源创建、编辑、删除、继续学习、打开资料的主要 mutation 细节。

建议测试：

- 在 `src/renderer/App.test.tsx` 或新增 hook 测试中模拟创建后选中。
- 模拟删除当前项目后回退到下一个有效项目。
- 模拟资源删除、继续学习和打开出关记录后，刷新、Toast、pending 行为仍保持一致。

## 14. 风险和缓解

| 风险 | 表现 | 缓解 |
| --- | --- | --- |
| App 拆分引发行为回归 | pending、confirm、toast 丢状态 | 先抽 hook，不改 UI；每拆一块跑 focused 测试 |
| 出关表单允许空进度导致 service 语义不清 | 误把空值写成 0 | DTO 区分 undefined/null/number，service 单测覆盖 |
| IPC output schema 工作量扩大 | 大量 DTO 要补 schema | 只覆盖本轮关键命令，其他命令保留旧路径 |
| CSS 分层造成视觉漂移 | 主题或布局突然变化 | legacy 过渡，迁移一类组件做一次截图 |
| 全局资源库 SQL 性能不足 | 大量 LIKE 查询变慢 | v1.5 先基础索引和分页，FTS 后置 |
| 境界定位继续膨胀 | 变成游戏系统 | 文档明确不做事项，所有突破反馈只绑定项目级进度 |
