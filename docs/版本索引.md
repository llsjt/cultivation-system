## 当前规则

1. 原根目录 `docs/需求文档.md` 和 `docs/实施方案.md` 不再作为当前入口。
2. 当前新增需求应进入 `versions/v1.7/需求分析.v1.7.md`。
3. 当前实施计划应进入 `versions/v1.7/实施方案.v1.7.md`。
4. 每个版本只保留一份需求分析和一份实施方案。
5. 开启新当前版本前，上一版本必须标明“已完成 / 已归档 / 冻结未完成”之一；若冻结未完成，必须在新版本需求分析中列出承接假设、后置项和风险。
6. “当前草稿”只能有一个版本；非当前草稿不得继续接收新增需求、范围调整或验收标准变更。
